# Source Priority

> 本文件定义新仓库中所有设计与实现决策的信息源优先级。

## 信息源层级

| 优先级 | 来源 | 说明 |
|--------|------|------|
| **P0** | Apple 官方 API 文档与 WWDC Session | 唯一权威技术参考 |
| **P1** | 本仓库 `docs/` 与 `specs/` 中的 Active 文档 | 项目内生效规范 |
| **P2** | 经 ADR 审批的例外决策 (`docs/adr/`) | 偏离 P0/P1 时的正式记录 |
| **P3** | 开发者在当次会话中的显式指令 | 仅对当次会话有效 |

## 禁止作为决策来源

1. 外部冻结归档目录中的历史文档——除非开发者在当次会话中显式要求"请检索历史归档"
2. AI 会话记忆或 Knowledge Item 中的旧组件定义、旧设计 token、旧文件路径
3. 非本仓库的任何第三方设计系统规范（除 Apple HIG 外）
4. 迁移治理过程文档（含审计报告、执行日志等）

## 冲突解决规则

- 当 P0 与 P1 冲突时，以 P0 为准，并更新 P1 使之对齐
- 当需要偏离 P0 时，必须提交 ADR 草稿并由人工审批后方可执行
- 任何信息源不在上表中的，默认视为不可用；如需引入，须先通过 ADR 流程

## 审计方式

- 每份 Active 文档须可追溯到 P0 或 P1 来源
- 代码审查时可要求作者标注决策依据所属层级
- 违反本优先级的内容一经发现应立即移除或降级至 ADR 待审
