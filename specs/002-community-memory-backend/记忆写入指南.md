# 社区记忆写入指南

> 本文档是记忆数据的录入规范。  
> 对应数据文件：`Resources/memories.json`  
> 对应数据模型：`Shared/Models/PointOfInterest.swift` → `MemoryPoint`
> 本次同步：2026-03-10（America/New_York）  
> 关联文档：`社区记忆库后端实装规划报告.md` Section 14（本文件为唯一规范真源）

---

## 1. 数据文件位置

```
YOM/
└── Resources/
    └── memories.json          ← 所有记忆点数据（只读，随 Bundle 打包）
```

---

## 2. JSON 结构规范

`memories.json` 的根结构：

```json
{
    "memoryPoints": [ ... ]
}
```

每个记忆点对象的完整字段如下：

### 2.1 MemoryPoint 字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | `String (UUID)` | ✅ | 唯一标识，格式 `XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX` |
| `year` | `Int` | ✅ | 记忆发生的年份，如 `1935` |
| `latitude` | `Double` | ✅ | 纬度坐标 |
| `longitude` | `Double` | ✅ | 经度坐标 |
| `distanceMeters` | `Int` | ✅ | 从参考基准点到该记忆点的预存静态距离（米），**仅用于列表初始排序**，不得作为实时距离展示给用户（实时距离由定位动态计算） |
| `nameByLanguage` | `Object` | ✅ | 多语言名称（见下方语言键） |
| `summaryByLanguage` | `Object` | ✅ | 多语言摘要，简短描述（1-2 句） |
| `storyByLanguage` | `Object` | ✅ | 多语言完整故事文本 |
| `unlockRadiusM` | `Double` | ✅ | 解锁半径（米），推荐 `50`-`100` |
| `tags` | `[String]` | ✅ | 标签数组，用于分类与筛选 |
| `media` | `[MemoryMedia]` | ✅ | 关联媒体资源数组（至少 1 个） |

### 2.2 语言键

所有 `ByLanguage` 字段使用以下键：

| 键 | 语言 |
|----|------|
| `en` | English |
| `zhHans` | 简体中文 |
| `yue` | 粤语 |

> ⚠️ 三种语言都必须填写，`en` 是 fallback 语言。

### 2.3 MemoryMedia 字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | `String (UUID)` | ✅ | 媒体唯一标识 |
| `type` | `String` | ✅ | 媒体类型，枚举值见下方 |
| `localAssetName` | `String` | ✅ | 本地资源文件名（不含扩展名） |
| `duration` | `Double?` | ❌ | 时长（秒），仅 `audio` / `video` 需要 |
| `thumbnailAssetName` | `String?` | ❌ | 缩略图资源名，仅 `video` / `ar` 需要 |

> ⚠️ Swift 模型 `MemoryMedia` 含 `memoryPointId` 字段，但 **JSON 中无需填写此字段**。`LocalMemoryRepository` 会在解码时以父级 `MemoryPoint.id` 自动回填。
>
> ⚠️ Phase 0 统一媒体扩展规范：`audio -> .m4a`、`video -> .mp4`、`ar -> .usdz`。历史 `.mp3` / `.mov` 资源需先转码或重命名后再入库。

### 2.4 媒体类型枚举

| `type` 值 | 说明 | 资源格式 |
|-----------|------|----------|
| `image` | 历史照片 | `.jpg` / `.png`（放入 `Assets.xcassets`） |
| `audio` | 口述录音 / 广播 | `.m4a`（Phase 0 统一规范，放入 Bundle） |
| `video` | 历史影像 | `.mp4`（Phase 0 统一规范，放入 Bundle） |
| `ar` | AR 3D 模型 | `.usdz`（放入 Bundle） |

## 3. 资源文件命名规范

```
memory_{地点关键词}_{年份}[_{序号}]
```

示例：

| 资源名 | 说明 |
|--------|------|
| `memory_market_street_1935` | 市场街 1935 年照片 |
| `memory_ocean_view_radio_1962` | 海景游乐场 1962 年广播录音 |
| `memory_civic_hall_1947_02` | 市政厅 1947 年第二张照片 |

## 5. 写入检查清单

在提交新记忆前，逐项确认：

- [ ] `id` 使用了全局唯一的 UUID（不能与现有条目重复）
- [ ] `latitude` / `longitude` 坐标精确到小数点后 4 位以上
- [ ] `nameByLanguage` 三种语言均已填写
- [ ] `summaryByLanguage` 三种语言均已填写（简短，1-2 句）
- [ ] `storyByLanguage` 三种语言均已填写（完整，3-5 句）
- [ ] `unlockRadiusM` 设定合理（市区建议 50m，郊区可放宽至 100m）
- [ ] `tags` 至少包含 1 个标签
- [ ] `media` 至少包含 1 个条目
- [ ] 每个 `media.localAssetName` 对应的文件已放入项目资源目录
- [ ] `audio` / `video` 类型已填写 `duration`（秒）
- [ ] 媒体扩展名符合 Phase 0 统一规范：`audio=.m4a`、`video=.mp4`、`ar=.usdz`
- [ ] JSON 格式合法（无尾逗号、引号正确闭合）
