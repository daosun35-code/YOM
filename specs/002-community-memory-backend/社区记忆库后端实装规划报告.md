# 社区记忆库本地化实装规划报告（Phase 0：不连接服务器）

> 文档日期：2026-03-09（America/New_York）  
> 当前约束：暂不连接任何后端服务；媒体内容全部内置在本地 App。  
> 目标：先把 Active / Passive 双模式在端内跑通，再进入后端化阶段。

## 0. 执行结论

本阶段采用 **Local-First 架构**：

1. 不调用远端 API，不做 APNs，不依赖云存储。
2. 记忆数据和媒体走本地资源（`Bundle` + 本地持久化）。
3. Active 模式与 Passive 模式都在端上闭环完成。
4. “已探索档案”和“记忆卡片”先保存为本地状态与本地文件。

## 0.1 强制实现约束

1. 新功能实现必须基于当前项目既有设计风格、组件层级和制作规范，不得另起一套视觉与交互语言。
2. 新功能必须使用 SwiftUI 落地，复用现有 DS 组件层（`DSColor` / `DSSpacing` / `.dsTextStyle` 等）。
3. 禁止绕过 DS 裸写 SwiftUI 或引入 UIKit 替代方案；新增页面/组件必须经由 DS Token 走样式。
4. 代码评审新增门禁：若新增功能绕过 DS 层裸写样式常量或引入 UIKit 视图，则视为不通过。
5. 所有新增 UI 必须走 `xcodebuildmcp` 辅助验证流程，以确保编译通过及架构合规。

## 1. 业务链路（本地闭环版）

### 1.1 Active 模式

1. 用户点击地图 `pin`，进入导航态。
2. App 在端上做“到达范围判定”（距离 <= `unlockRadius`，可加停留秒数）。
3. 达标后解锁记忆详情，按本地媒体类型渲染：`image` / `audio` / `video` / `ar(usdz)`。
4. 用户点“体验完成”后写入本地档案为 `已探索`。
5. 端上生成分享卡片图片，支持保存相册和系统分享。

### 1.2 Passive 模式

1. 首次启用“附近记忆提醒”时请求通知权限与定位权限。
2. 用本地记忆点计算“最近监控点集合”，注册设备端地理围栏。
3. 用户靠近时由本地通知触发文案，点击后直达 Active 链路。

## 2. 端内架构设计

```text
[iOS App]
  -> LocalMemoryRepository (本地记忆数据)
  -> LocalMediaCatalog (Bundle 媒体索引)
  -> GeoEngine (距离判定 + geofence 管理)
  -> ExplorationStore (已探索记录)
  -> CardRenderer (端上卡片渲染)
  -> NotificationOrchestrator (本地通知编排)
```

### 2.1 模块职责

1. `LocalMemoryRepository`：加载本地记忆数据（JSON/SQLite/CoreData）。
2. `LocalMediaCatalog`：将 `memoryId -> 本地媒体资源` 建索引。
3. `GeoEngine`：负责最近点计算、进入范围判定、地理围栏轮换。
4. `ExplorationStore`：记录探索状态、完成时间、来源（active/passive）。
5. `CardRenderer`：把记忆内容渲染为图片（卡片模板）。
6. `NotificationOrchestrator`：注册本地提醒、去重与冷却。

## 3. 本地数据模型（建议）

> 设计原则：现有 `Shared/Models/PointOfInterest.swift` 已包含 `id`, `year`, `coordinate`, 多语言 `name`/`summary`，是地图 pin 的真源。新模型必须与之统一，避免同一地点记忆出现两份不关联的 model。

**方案：扩展 `PointOfInterest` -> `MemoryPoint`**

1. `MemoryPoint`（扩展自 `PointOfInterest`）
`id, year, coordinate, nameByLanguage, summaryByLanguage, story, unlockRadiusM, tags, media[]`
2. `MemoryMedia`
`id, memoryPointId, type(image/audio/video/ar), localAssetName, duration, thumbnailAssetName`
3. `ExplorationRecord`
`id, memoryPointId, exploredAt, source(active/passive), cardLocalPath`
4. `WatchRegion`
`memoryPointId, center, radiusM, enabledAt, cooldownUntil`

> 建议：`MemoryPoint` 与 `MemoryMedia` 放只读本地包（Bundle JSON）；`ExplorationRecord` 与 `WatchRegion` 放可写本地存储（SwiftData 优先，iOS 17+ 原生支持，备选 CoreData）。

## 4. 状态机与判定规则

### 4.1 状态转换图

```
IDLE --(tap pin)--> NAVIGATING --(in range)--> IN_RANGE --(dwell ok)--> UNLOCKED --(完成)--> EXPERIENCED --(归档)--> ARCHIVED
  ^                    |  ^                      |                        |
  |                    |  |                      |                        |
  +---(cancel/fail)----+  +---(exit range)-------+                        |
  +-----------------------------------------------(exit without finish)---+
```

### 4.2 状态转换清单

| 转换 | 触发条件 | 副作用 |
| --- | --- | --- |
| `IDLE -> NAVIGATING` | 用户点击 pin 并确认导航 | 创建 `NavigationSession` |
| `NAVIGATING -> IN_RANGE` | `distance <= unlockRadius` | 启动 dwell 计时器 |
| `IN_RANGE -> UNLOCKED` | `dwellAccumulatedSeconds >= 10` | 解锁详情页 |
| `UNLOCKED -> EXPERIENCED` | 用户点击"体验完成" | -- |
| `EXPERIENCED -> ARCHIVED` | 自动 | 写入 `ExplorationRecord` |
| `NAVIGATING -> IDLE` | 用户取消导航 / 定位失败 | 销毁 `NavigationSession` |
| `IN_RANGE -> NAVIGATING` | 用户离开 `unlockRadius` | 重置 dwell 计时器 |
| `UNLOCKED -> IDLE` | 用户退出详情未完成体验 | 不写入 `ExplorationRecord` |
| 任意 -> `ERROR` | 定位服务不可用 / 权限被撤销 | 弹出恢复引导 |

### 4.3 判定规则

1. `ARCHIVED` 幂等，同一记忆重复完成不重复建档。
2. 解锁判定采用"双条件"：`distance <= unlockRadius` + `dwellAccumulatedSeconds >= 10`。
3. **dwell 采用累计停留时间**（非连续）：在 `unlockRadius` 内的采样点按时间戳累加，短暂跳出（< 3s）不重置计数器，以容忍 GPS 城市峡谷效应下的信号抖动。
4. Passive 触发后，若该记忆已 `ARCHIVED`，默认不再重复通知。

## 5. Active 模式实现细节（无服务端）

1. 点击 pin：创建本地 `NavigationSession`（内存态即可）。
2. 导航期间持续取位置，实时算到目标点距离。
3. 达标触发本地解锁事件，详情页从 `LocalMemoryRepository` 读取。
4. **详情页复用现有 `Features/Retrieval/RetrievalView.swift`**，在其基础上扩展媒体渲染能力：
   - `image`：本地图片资源
   - `audio/video`：本地文件 URL + 内嵌播放器
   - `ar`：本地 `usdz` + Quick Look
5. 体验完成后写 `ExplorationRecord`，并触发本地卡片渲染。

## 6. Passive 模式实现细节（无服务端）

### 6.1 权限策略

1. 首屏先解释“附近记忆提醒价值”，再触发权限请求。
2. 权限顺序建议：通知权限 -> 定位权限（含后台用途文案）。

### 6.2 地理围栏策略（CLMonitor，iOS 17+）

> API 选型：`CLCircularRegion` 在 iOS 17 已 deprecated。本项目统一使用 `CLMonitor` + `CLCircularGeographicCondition`（async/await 模式），事件通过 `AsyncSequence` 接收，与 SwiftUI `task {}` 生命周期天然契合。

1. 设备同时监控数量上限为 **20**（超出时 `CLMonitor` 发出 `conditionLimitExceeded`），需仅保留"最近 N 个"（N <= 20）。
2. 每次位置显著变化后，重算最近集合并替换监控列表。
3. 对同一记忆设置冷却窗口（例如 24h）避免频繁打扰。
4. 最小有效半径建议 >= 200m（Apple 官方建议），密集城区可酌情缩至 100m。
5. 已知风险：iOS 18 早期版本 exit 事件可能在远处误触发，需加距离二次校验容错。

### 6.3 通知触发与跳转

1. 进入围栏后下发本地通知（`UNUserNotificationCenter`）。
2. payload 带 `memoryId`，点击后路由到对应记忆详情。
3. 进入详情后复用 Active 的解锁与归档逻辑。

## 7. 记忆卡片（本地生成）

1. 输入：标题、摘要、地点名、时间、封面图、本地配色模板。
2. 输出：卡片图片文件（App 沙盒）。
3. 分享：
保存相册（`PHPhotoLibrary`）；
系统分享（`UIActivityViewController`）。

## 8. 本地方案边界（已知限制）

1. 无法做远程运营（推送 AB、服务端实时策略）。
2. 设备丢失会丢本地探索档案（除非后续加 iCloud/后端同步）。
3. 地理围栏数量有限，长尾记忆点需动态轮换而非全量常驻。

## 9. 分阶段实施（仅本地）

### Phase 0A（1 周）

1. 落地本地记忆数据模型（`MemoryPoint` 扩展自 `PointOfInterest`）与媒体索引。
2. 跑通 Active：导航 -> 解锁 -> 详情媒体播放（复用 `RetrievalView`）。

**DoD（验收标准）**：
1. 从本地 JSON 加载 >= 3 条 `MemoryPoint`，地图 pin 正常渲染。
2. 点击 pin -> 导航态 -> 到达范围内 -> 详情页展示标题/年份/摘要/媒体。
3. UI Test 断言：`memory_detail_title` 存在且文本匹配。

### Phase 0B（1 周）

1. 落地探索归档与档案页展示。
2. 落地卡片本地渲染 + 保存相册 + 系统分享。

**DoD（验收标准）**：
1. 体验完成后 `ExplorationRecord` 持久化，重启 App 仍可见。
2. 卡片渲染输出 >= 1 张图片，可保存至相册。
3. 系统分享 Sheet 弹出且可交互。

### Phase 0C（1 周）

1. 落地 Passive：`CLMonitor` geofence + 本地通知 + 点击跳转。
2. 完成去重/冷却与边界回归测试。

**DoD（验收标准）**：
1. Passive 模式注册 >= 1 个 `CLMonitor` 条件，进入围栏后收到本地通知。
2. 同一记忆 24h 内不重复通知。
3. 已 `ARCHIVED` 的记忆默认不触发通知。

## 10. 阶段验收与自动化查验方案 (QA & Acceptance Testing)

为确保 Local-First 架构及 UI 渲染质量，每个阶段（Phase 0A, 0B, 0C）结束后均需进行相应的自动化与半自动化检测。推荐的 XCTest / Snapshot 测试方案如下：

### 10.1 Unit Tests (XCTest)
用于验证纯逻辑组件，如距离判定、停留时间计算：
- `UnlockEvaluatorTests`: 模拟不同的 GPS 坐标流，测试 `distance <= unlockRadius` 是否正确触发。
- `DwellTimerTests`: 模拟累计停留时间累加及异常跳出剔除，确保 >= 10s 的阈值计算准确。
- `LocalMemoryRepositoryTests`: 从本地 JSON 中加载并序列化 `MemoryPoint`，防止格式错误导致崩溃。

### 10.2 Snapshot Tests (swift-snapshot-testing)
用于防止 UI 组件发生视觉退化，确保 DS Token 体系未被破坏：
- 对 `RetrievalView` 及其内的多媒体组件（CardRenderer 生成的图）等核心 View 使用 Snapshot 测试。
- 强制所有新增页面/UI必须提供至少一个浅色/深色模式下的 Snapshot 用例，并配合 `xcodebuildmcp` 进行验证。

### 10.3 UI Tests (XCUITest)
用于自动化用户操作流及页面跳转验证：
- 配合 Xcode 提供 Mock Location 服务（GPX 文件注入坐标）。
- 启动 App 后验证 `NAVIGATING -> IN_RANGE -> UNLOCKED -> EXPERIENCED` 这一完整闭环状态机的状态流转及 UI 元素更新。
- 包含 Accessibility Identifier 断言，确保核心按钮（如"体验完成"）可达且行为符合预期。

## 12. 后续迁移到后端（保留位）

当你准备连服务器时，沿用当前端内模型做平滑迁移：

1. `LocalMemoryRepository` 升级为 `Repository` 协议：本地实现 + 远端实现可切换。
2. `ExplorationStore` 新增同步队列，离线写本地、在线回传服务端。
3. 通知从本地触发逐步过渡到“本地 + APNs 混合模式”。

## 14. 附录：社区记忆写入指南

> 本指南为记忆数据的录入规范。对应数据文件：`Resources/memories.json`，对应模型：`Shared/Models/PointOfInterest.swift` → `MemoryPoint`。

### 14.1 数据文件位置

```text
YOM/
└── Resources/
    └── memories.json          ← 所有记忆点数据（只读，随 Bundle 打包）
```

### 14.2 JSON 结构规范

`memories.json` 的根结构：

```json
{
    "memoryPoints": [ ... ]
}
```

每个记忆点对象的完整字段如下：

#### A. MemoryPoint 字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | `String (UUID)` | ✅ | 唯一标识，格式 `XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX` |
| `year` | `Int` | ✅ | 记忆发生的年份，如 `1935` |
| `latitude` | `Double` | ✅ | 纬度坐标 |
| `longitude` | `Double` | ✅ | 经度坐标 |
| `distanceMeters` | `Int` | ✅ | 记忆点之间的默认排序距离（米），用于列表展示 |
| `nameByLanguage` | `Object` | ✅ | 多语言名称（见下方语言键） |
| `summaryByLanguage` | `Object` | ✅ | 多语言摘要，简短描述（1-2 句） |
| `storyByLanguage` | `Object` | ✅ | 多语言完整故事文本 |
| `unlockRadiusM` | `Int` | ✅ | 解锁半径（米），推荐 `50`-`100` |
| `tags` | `[String]` | ✅ | 标签数组，用于分类与筛选 |
| `media` | `[MemoryMedia]` | ✅ | 关联媒体资源数组（至少 1 个） |

#### B. 语言键

所有 `ByLanguage` 字段使用以下键：

| 键 | 语言 |
|----|------|
| `en` | English |
| `zhHans` | 简体中文 |
| `yue` | 粤语 |

> ⚠️ 三种语言都必须填写，`en` 是 fallback 语言。

#### C. MemoryMedia 字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | `String (UUID)` | ✅ | 媒体唯一标识 |
| `type` | `String` | ✅ | 媒体类型，枚举值见下方 |
| `localAssetName` | `String` | ✅ | 本地资源文件名（不含扩展名） |
| `duration` | `Double?` | ❌ | 时长（秒），仅 `audio` / `video` 需要 |
| `thumbnailAssetName` | `String?` | ❌ | 缩略图资源名，仅 `video` / `ar` 需要 |

#### D. 媒体类型枚举

| `type` 值 | 说明 | 资源格式 |
|-----------|------|----------|
| `image` | 历史照片 | `.jpg` / `.png`（放入 `Assets.xcassets`） |
| `audio` | 口述录音 / 广播 | `.mp3` / `.m4a`（放入 Bundle） |
| `video` | 历史影像 | `.mp4` / `.mov`（放入 Bundle） |
| `ar` | AR 3D 模型 | `.usdz`（放入 Bundle） |

### 14.3 资源文件命名规范

```text
memory_{地点关键词}_{年份}[_{序号}]
```

示例：

| 资源名 | 说明 |
|--------|------|
| `memory_market_street_1935` | 市场街 1935 年照片 |
| `memory_ocean_view_radio_1962` | 海景游乐场 1962 年广播录音 |
| `memory_civic_hall_1947_02` | 市政厅 1947 年第二张照片 |

### 14.5 写入检查清单

在提交新记忆前，逐项确认：

- [ ] `id` 使用了全局唯一的 UUID（不能与现有条目重复）
- [ ] `latitude` / `longitude` 坐标精确到小数点后 4 位以上
- [ ] `nameByLanguage` 三种语言均已填写
- [ ] `summaryByLanguage` 三种语言均已填写（简短，1-2 句）
- [ ] `storyByLanguage` 三种语言均已填写（完整，3-5 句）
- [ ] `unlockRadiusM` 设定合理（市区建议 50m，郊区可放宽至 100m）
- [ ] `tags` 至少包含 1 个标签
- [ ] `media` 至少包含 1 个条目
- [ ] 每个 `media.localAssetName` 对应的文件已放入项目资源目录
- [ ] `audio` / `video` 类型已填写 `duration`（秒）
- [ ] JSON 格式合法（无尾逗号、引号正确闭合）
